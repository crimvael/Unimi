Exercise 3: Testing Algebra.

A group is an algebraic structure consisting of a set S together with an operation · that combines any two of its elements to form a third element. To qualify as a group, the set S and the operation · must satisfy a few conditions called group axioms, namely

  • closure
  • associativity
  • identity
  • invertibility.

For example, the set Z (the integers with sign) with the + operator is a group but with the * is not since inverting the operation break the closure property.

Write the Group class as a template for the real group algebraic structure. such a class must be general, in the sense that the operations and the sets are not defined a priori. The Group container will be instantiated to realize the concrete group.

Test driven development will be used to test that the concrete groups respect the group axioms (closure, associativity, identity and invertibility), i.e., that they really are groups.

Test your implementation with the following examples (S is the set, · is the operation, and i the identity):

Groups

  • S = {True, False}, · = or, i = False
  • S = Z, · = +, i = 0
  • S = Z4 = {0, 1, 2, 3}, · = * where a*b=(a*b)%4, i = 1
  • S = {a}*, ·: {a}*×{a}* --> {a}* where x1·x2 is x1 if #x1 > #x2, x2 otherwise, i = the empty string

Note infinite sets can be implemented through iterators; property checks on infinite sets should be on a finite subset.
