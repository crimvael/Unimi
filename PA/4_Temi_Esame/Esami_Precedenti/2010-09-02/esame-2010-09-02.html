Exercise 1: Text on Nine Keys (T9).

T9, which stands for Text on 9 keys, is a predictive text technology for mobile phones. T9's objective is to make it easier to type text messages. It allows words to be entered by a single key press for each letter. It combines the groups of letters on each phone key with a fast-access dictionary of words. It looks up in the dictionary all words corresponding to the sequence of key presses and orders them by frequency of use.

From our point of view T9 is just a mechanism to map the 26 alphabetic characters to 9 (indeed only 8 since the key 1 is reserved for punctuation and figures) figures. Of course, decoding is not univocally defined (to a single figure are associated from 3 to 4 letters) but its compression rate is quite elevate (less than 33%) and its implementation quite easy.

Given the dictionary without frequencies dictionary.txt write a function that calculates (and prints) all the possible translations for the sentences contained in the file t9's texts.txt. The dictionary contains all English words one a row without punctuation signs, accents, dashes, contractions or apostrophes (just lowercase letters). The coded sentences are without punctuation signs. The test file contains a coded sentence each row and the sentences do not contain punctuation signs. The program must print all the possible decoding for each sentence before passing to decode the next sentence. The dictionary will permit you to have meaningful translations skimming out all the generated words that are not in the dictionary.

Note that, brute force approaches are not appreciated (if not considered wrong at all) as well as mismatched translations (i.e., sentences with words that do not match the length of the coded word or sentences with more or less words of the coded version). The use of ìtertools or similar packages is forbidden.

Exercise 2: Text on Nine Keys (T9) with Frequencies.

Repeat exercise 1 by using the dict-frequency.txt dictionary. This dictionary differs from the previous because each word has associated a frequency. Frequencies represents the number of occurrences in some predefined (huge) texts, frequencies are are per billion.

Frequencies should be used to select a word out of those associated to a coded word. The output for this exercise is one translation for each listed sentence with ambiguities solved by frequencies. Do not worry if the sentence is meaningless it is correct if it is a possible translation.

Note that exercises 1 and 2 almost do the same thing so reuse is awarded redoing it completely will be penalized.

Exercise 3: Usage Memory.

Under Linux a lot of information about a process are kept during its execution. Any information about a process, included identity and used memory, can be retrieved by parsing the content of the pseudo-file:

/proc/«pid»/status

where the «pid» is the process identifier for the corresponding running process. Such a file is dynamically updated and can be used to measure the memory consumption (size) for a piece of python code, e.g., for a method call.

Write a decorator (named @memory_consumption) that when applied to a function will measure (and print on the screen) the memory consumption of the calls to such a function. Try out this decorator on the main function for the exercise 1.

Note, that package os is your friend.
