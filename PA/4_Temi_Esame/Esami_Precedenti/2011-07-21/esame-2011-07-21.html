Exercise 1: Wandering around Europe.

As everyone knows Europe is a collection of countries close to each other; each country has a pool of neighboring countries whose inhabitants have to cross to reach other countries. Basically when you plan a car trip around Europe it is a common issue to determine how many countries it is necessary to cross (in order to get the right currencies and all your electric equipment working at every stop).

To help planning your trip you have to write a function crossing that given a country name and n (the desired number of steps) gives you the set of countries you can reach crossing n countries starting from the given country. Note that:

  1. in zero steps you can't exit from the starting country;
  2. in one step you get only in the border countries;
  3. the path must always be minimal, e.g., from Italy you can get in Belgium crossing Switzerland and Germany or only France the second is the one to choose;
  4. you can't cross a country twice or more or get back, e.g., Italy → Switzerland → Italy or Italy → Switzerland → France are wrong.

Since Europe is tightly coupled to Asia in the exercise exclude Kazakhstan, Georgia, Azerbaijan, Cyprus and Turkey from Europe, i.e., consider Europe as a group of islands separated from Asia in the listed countries (that could be considered as ocean if you like). To help those with problems in geography this is a political map of Europe to be used.

Hint define a generator border, used by crossing, that given the starting country at each step provides the set of border countries of the border computed in the previous step.

Exercise 2: Are You Wandering Correctly around Europe?

Since vacations are an important matter we cannot do our work wrong. It is time to validate exercise 1 against errors. Use test-driven development to write a class to run tests on the previous exercise.

Hint. Some properties that should be valid in the previous exercise are:

  1. all countries listed as neighbors of another country must list such a country as a neighbor
  2. a path can be followed in the two directions, i.e., if you get in c₁, c₂ and c₃ from a country c in n steps then in n steps from either c₁, c₂ and c₃ you must get in c.

These are just a couple of necessary test not the only possible but represents a good minimal pool of tests to consider your exercise correct.

Exercise 3: Sleep Sort.

Recently on the Internet is possible to see a novel approach to sorting named sleep sort that promises linear time complexity. Such an approach rely on the sleep system call to sort the numbers: each number is dealt as a thread which is suspended as many milliseconds as the number itself and when resumed it will just print its number. The scheduler will play the magic by resuming the threads in the correct order.

The exercise consists of coding the sleep sort algorithm as a python function that gets the numbers to be sorted and spawns the threads according to the presented algorithm.
